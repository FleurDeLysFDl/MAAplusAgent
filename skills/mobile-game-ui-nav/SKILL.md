这份技能总结的是国内主流二次元/放置养成类手游（无期迷途、阴阳师、FGO这类）
共有的UI设计套路。不同游戏具体样式不一样，但结构高度相似，遇到没见过的
界面先往这几类模式上套，能少走很多摸索的弯路。

## 常见界面模式

**主城/大厅界面**：一堆功能建筑/图标入口平铺，通常在屏幕四周分布（比如
"危机管理""整备中心""管理局""采购办""禁闭者"这种）。这是信息密度最高的
一屏，优先把这些入口一个个点开看看，比钻进某一个功能的三级页面更容易
发现新东西。

**顶部tab+左侧列表+右侧详情**（公告/资讯类界面的典型布局）：顶部几个tab
（"系统公告""活动公告""狄斯资讯"），左边是列表，点一条左边列表项，右边
详情区跟着换内容。列表项之间切换属于同一个节点内的状态变化，不是新界面；
换tab才算真的换了功能模块。

**角色养成类界面**：进入后一般是"角色列表"（一排头像/卡片），点进某个
角色是"角色详情"，详情页里还会有一排二级tab（"信息""技能""培养""烙印"
"档案"之类）。这一层级很容易产的新界面数量很多，但大多是同一个角色养成
系统的不同切面，摸清一个角色的详情结构后，其它角色的对应页面大概率长
得差不多（同一套UI换了角色立绘和数值），不需要每个角色都点全所有tab。

**兑换/礼包/商城类界面**：清一色是"物品图标+数量+一个动作按钮（前往/兑换/
购买）"这种列表项重复排列。这类界面的"新颖度"主要体现在整个列表页面
本身（比如"样本兑换"跟"密令兑换"是两个不同页面），而不是列表里的每一
条目——同一个兑换页面里的每一行不需要当成独立目标去探索。

**任务/进度追踪类界面**：进度条+"（0/1）"这种完成度文案+"前往"按钮。
同一个任务从"未完成"变成"冷却中"/"已完成"是页面内的状态变化，不是新
界面，即使OCR文字集合因此有明显变化也不代表真的换了功能模块（工具层
已经有图片diff兜底处理这种情况，但Agent自己判断"这算不算新发现"时也
该有这个概念，不要把同一个列表的不同状态快照都数成新发现）。

## 交互坑点

**"充值"一类敏感词经常是误报，但仍然必须照章停止**：邮箱/公告里常见
"关于打击违规充值和不诚信退款的公告"这种官方声明，纯文本扫描会命中
"充值"这个敏感词，实际跟真正的支付页面毫无关系。护栏是在MCP Server层
按OCR全文做关键词扫描的，做不到理解语境，所以确实会有误报。但操作
规则要求"命中敏感警告立刻停止调用Finish汇报"是不可以为了"这次可能是
误报"而不遵守的——正确做法是照做，在Finish的汇报里如实说明命中的是
哪个界面、判断可能是误报，而不是想办法绕过警告继续点击当前界面。

**关闭弹窗/浮层优先用press_back，不要瞎点空白区域**：绝大多数浮层/弹窗
用安卓返回键就能关，比自己猜坐标点空白处关闭按钮可靠得多。press_back
连续两次在同一个节点上都没有效果（screenshot()返回的known_ineffective
里能看到），才值得考虑get_raw_image找×号或者点弹窗外空白区域。

**图标类按钮OCR基本认不出来，别瞎猜坐标**：设置齿轮、耳机音效、地球
（服务器/语言）、登出箭头这类纯图标按钮，OCR候选框里通常没有对应文字，
click()工具会因为坐标不在候选框范围内直接拒绝执行。这种情况老老实实
调get_raw_image看原图再click_on_image，不要凭经验猜一个坐标去撞。

**同一套UI在不同分辨率/不同状态下坐标会变，不要迁移经验坐标**：这个
技能只讲"这是什么类型的界面、大概怎么走"，具体某个按钮在(x,y)这种精确
坐标信息不在这里——那是exploration_memory记录的known_actions该管的事，
每个游戏、每次探索都要重新用OCR候选框确认，不能凭这份技能里的经验瞎猜
数值坐标。
